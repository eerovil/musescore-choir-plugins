Read this pdf and for each LINE (printed system) in the score, give the lyrics in
JSON format:
[
    {
        "measure_start": 1,   # First measure of this line. If a measure number is
                              # printed at the start of the system use it; else null
                              # (it will be inferred from the line's position).
        "lyrics": [
            {
                "staff_number": 1,        # Which printed staff, counted 1 = top of
                                          # the system to bottom.
                "position": "above",      # "above"/"below" when two divisi voices
                                          # share one staff and sing DIFFERENT words;
                                          # omit (or null) if the staff has one line.
                "text": "this is the text the te-nors sing",
                "parts": []               # LEAVE EMPTY. Only fill to override the
                                          # mapping (see below).
            },
            {
                "staff_number": 2,
                "text": "ly-rics for the bass",
                "parts": []
            }
        ]
    }
]

Notes:
- ALWAYS include "parts": [] (empty) in every lyric. Leave it empty by default — the
  text is then mapped automatically to the right voice(s) from "staff_number" and
  "position". Only put something in "parts" to correct a wrong mapping (see below).
- Give each line's text by "staff_number" (top staff = 1) and, for a staff shared by
  two divisi voices that sing different words, a "position" ("above"/"below"). If the
  two voices on a staff sing the SAME words, give one line and omit "position".
- Split each word into syllables with hyphens, e.g. "ly-ric". Give the syllables as
  they are actually sung; don't worry about matching the exact note count
  (melismas/slurs are handled separately) and don't add padding or melisma markers.
- A word that continues onto the next line ends with a trailing hyphen (e.g. "Su-ku-")
  and the next line starts with the rest of it ("pol-vis-ta …").
- Only verse 1 is imported. Skip later verses and editorial/ossia notes.

MANUAL OVERRIDE (when the automatic mapping is wrong — e.g. a part is omitted on some
lines, or an extra voice like an ossia T3 is printed on top so the staff numbering
doesn't match):
- Fill the line's "parts" array to force exactly which voices get the text. It wins
  over staff_number/position. You can use:
    * part NAMES — "parts": ["T1", "T2"]  (matches the staff labels in the score;
      robust even when staves are reordered) — PREFERRED for overrides.
    * output staff IDs — "parts": [1, 2]   (the output staves sorted high-to-low:
      1, 2, 3, …).
- List several entries when staves differ; list several names/ids in one "parts" when
  they share the SAME words.
