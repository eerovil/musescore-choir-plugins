Read this pdf and for each LINE (printed system) in the score, give the lyrics in
JSON format:
[
    {
        "measure_start": 1,   # First measure of this line. If a measure number is
                              # printed at the start of the system use it; else null.
        "lyrics": [
            {
                "parts": ["T1", "T2"],   # Which voice part(s) sing this text. Use the
                                         # part LABEL printed at the left of the staff
                                         # (e.g. S, A, T1, T2, T3, B). List several
                                         # when they share a staff and sing the SAME
                                         # words; give separate entries when they differ.
                "text": "this is the text the te-nors sing"
            },
            {
                "parts": ["B"],
                "text": "ly-rics for the bass"
            }
        ]
    }
]

Notes:
- Identify voices by the part label at the left of each staff. The output score has one
  staff per part (T1, T2, T3, B, …) and matches these names exactly, so addressing parts
  by name is reliable even when a part is omitted on some lines or printed in an unusual
  order (e.g. an extra T3 placed on top).
- Split each word into syllables with hyphens, e.g. "ly-ric". Give the syllables as they
  are actually sung; don't worry about matching the exact note count (melismas/slurs are
  handled separately in the score) and don't add padding or melisma markers.
- A word that continues onto the next line ends with a trailing hyphen (e.g. "Su-ku-")
  and the next line starts with the rest of it ("pol-vis-ta …").
- Only verse 1 is imported. Skip later verses and editorial/ossia notes.
- If you are unsure of a part name, you may instead give "parts" as output staff ids
  (integers, top to bottom: 1, 2, 3, …). Names are preferred.
